# SEIF: Self-Evolving Reinforcement Learning for Instruction Following

This project implements an iterative self-training pipeline for instruction-following models, consisting of two roles that mutually improve each other:

- **Instructor**: Learns to generate diverse, challenging multi-constraint instruction prompts.
- **Follower**: Learns to follow multi-constraint instructions generated by the Instructor.

The iterative loop alternates between training Instructor and Follower, progressively improving both models.

## 1. Project Overview

The system is based on the Self-IF (Self-Instruction Following) framework. In each iteration:

1. The **Instructor** generates multi-constraint instructions for seed questions.
2. The **Follower** learns to follow these constraints.
3. The trained Follower serves as the evaluator for the next Instructor's training.

The key insight is that by using the latest Follower as the judge/filter/answer model, the Instructor learns to generate instructions that the Follower **cannot yet** follow well — forcing the Instructor to generate progressively harder and more diverse constraints.

## 3. Key Files

### 3.1 Reward Function — `examples/reward_function/instruction_reward.py`

This file contains the reward functions for both Instructor and Follower training. **Both implementations exist in the same file — one is active, one is commented out.** Comment/uncomment the appropriate blocks to switch between roles.

**Active (Instructor Training) — top of file, uncommented:**

Uses three vLLM model endpoints for verification:

| Model | Port | Role |
|-------|------|------|
| `ANSWER_VLLM` | 55222 | Generates answers to verify constraint feasibility |
| `FILTER_VLLM` | 55333 | Detects logically conflicting constraints |
| `JUDGE_VLLM` | 55111 | Judges whether each constraint is followed |

Key functions: `instruction_compute_score()`, `instruction_val_compute_score()`.

**Commented out (Follower Training) — lines 1–~1040, fully commented:**

Uses `instructions_registry` with per-constraint `check_following()` methods. Key function: `instruction_compute_score()` at line ~82 (commented). Directly evaluates the Follower's response against each constraint in the generated `q_Tk.parquet` data.

### 3.2 Dataset Utility — `verl/utils/dataset.py`

This file contains the dataset class for both Instructor and Follower training. **Both implementations exist in the same file — one is active, one is commented out.** Comment/uncomment the appropriate blocks to switch between roles.

**Active (Instructor Training) — `RLHFDataset` class at line ~220, uncommented:**

Uses `TASK_DESCRIPTION_TEMPLATE_HARD` or `TASK_DESCRIPTION_TEMPLATE_SOFT` to construct multi-constraint prompts from seed questions. Training data: `data/seed.parquet`.

**Commented out (Follower Training) — `RLHFDataset` class at lines 375–483, fully commented:**

Reads directly from the generated parquet file (e.g., `q_T1.parquet`). The parquet contains `instruction_id_list`, `kwargs`, and `constraints` columns generated by `chuti.py`. The `ground_truth` stores these fields for the reward function to evaluate.

### 3.3 Configuration Files

| File | Purpose | Key Settings |
|------|---------|--------------|
| `examples/config.yaml` | Instructor training | `train_files: data/seed.parquet`, GRPO algorithm, `n=5` rollouts per prompt |
| `examples/config1.yaml` | Follower training | Uses `q_Tk.parquet`, same algorithm settings |

### 3.4 Constraint Generation — `data/Self-IF-Self-QWEN-7B/chuti.py`

Generates multi-constraint instructions for seed questions using the Instructor model. Output format: `q_T{turn}.parquet`.

## 4. Iterative Training Workflow

The following diagram shows the complete iterative loop (using Qwen2.5-7B-Instruct as example):

```
Round T (T ≥ 1):
┌─────────────────────────────────────────────────────────┐
│  STEP 1: Train Instructor (I_T)                         │
│  ─────────────────────────────────────────────────────  │
│  Use vLLM servers (ANSWER, FILTER, JUDGE) with I_{T-1} │
│  (or base model for T=1)                                │
│                                                         │
│  Training data: data/seed.parquet                      │
│  Config: examples/config.yaml                           │
│  dataset.py: uncomment Instructor RLHFDataset          │
│  reward: uncomment Instructor reward functions          │
│  Output: Instructor checkpoint I_T                      │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  STEP 2: Serve Instructor (I_T) for data generation     │
│  ─────────────────────────────────────────────────────  │
│  Start vllm_chuti.sh with I_T                          │
│  Run chuti.py to generate q_T.parquet                  │
│                                                         │
│  e.g., data/Self-IF-Self-QWEN-7B/q_T1.parquet          │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  STEP 3: Train Follower (F_T)                           │
│  ─────────────────────────────────────────────────────  │
│  Use vLLM verifier server with F_{T-1}                 │
│  (or base model for T=1)                                │
│                                                         │
│  Training data: q_T.parquet                            │
│  Config: examples/config1.yaml                         │
│  dataset.py: uncomment Follower RLHFDataset           │
│  reward: uncomment Follower reward functions           │
│  Output: Follower checkpoint F_T                        │
└─────────────────────────────────────────────────────────┘

Round T+1:
  → Replace ANSWER/FILTER/JUDGE models with F_T
  → Train next Instructor I_{T+1}
  → Serve I_{T+1}, generate q_{T+1}.parquet
  → Replace verifier with F_T
  → Train next Follower F_{T+1}
  → Repeat...
```

## 5. Detailed Steps

### 5.1 First-Time Setup

Before starting the iterative training, prepare the following:

1. **Prepare seed data** — Ensure `data/seed.parquet` contains seed questions.
2. **Base models** — Download the base model (e.g., `Qwen/Qwen2.5-7B-Instruct`).
3. **Install dependencies** — Ensure `verl`, `vllm`, `ray`, and other dependencies are installed.
4. **Environment** — Set up GPU resources. Each vLLM server and training run requires specific GPU allocations (see vLLM server scripts).

---

### 5.2 Round 1: Train Instructor

#### Step 1.1: Start vLLM servers for evaluation

Use the **base model** (or any pre-trained model) as the initial evaluator:

```bash
# Terminal 1: Answer model (port 55222)
bash vllm_answer.sh

# Terminal 2: Filter model (port 55333)
bash vllm_filter.sh

# Terminal 3: Judge model (port 55111)
bash vllm_judge.sh
```

> **Important**: Verify all three vLLM servers are running and responsive before proceeding.

#### Step 1.2: Enable Instructor reward in instruction_reward.py

In `examples/reward_function/instruction_reward.py`, **ensure the Instructor reward block is uncommented** (top of file, the three vLLM-based functions) and **comment out the Follower reward block** (lines 1–~1040).

#### Step 1.3: Enable Instructor dataset in dataset.py

In `verl/utils/dataset.py`, **uncomment the Instructor `RLHFDataset` class** (around line ~220) and **comment out the Follower `RLHFDataset` class** (lines 375–483).

#### Step 1.4: Configure training launcher

Edit `examples/qwen.sh` to point to the initial model:

```bash
MODEL_PATH=Qwen/Qwen2.5-7B-Instruct  # or path to your base model
SAVE_PATH=/path/to/save/I_T1
```

#### Step 1.5: Launch Instructor training

```bash
bash examples/qwen.sh
```

This trains the Instructor using GRPO on `data/seed.parquet`. The Instructor learns to generate diverse, challenging multi-constraint instructions.

**Output**: Instructor checkpoint (e.g., `/path/to/I_T1/`).

---

### 5.3 Round 1: Train Follower

#### Step 1.6: Start vLLM server for Instructor

After Instructor training completes, start the Instructor model as a service for data generation:

```bash
# In a new terminal
bash data/Self-IF-Self-QWEN-7B/vllm_chuti.sh
```

This serves the Instructor at the configured endpoint. The `chuti.py` script calls this endpoint to generate constraints.

#### Step 1.7: Generate training data with Instructor

```bash
cd data/Self-IF-Self-QWEN-7B
python chuti.py
```

This generates multi-constraint instructions for each seed question and saves the result as `q_T1.parquet`.

#### Step 1.8: Enable Follower reward in instruction_reward.py

In `examples/reward_function/instruction_reward.py`, **uncomment the Follower reward block** (lines 1–~1040, the `instructions_registry`-based `instruction_compute_score()`) and **comment out the Instructor reward block** (top of file).

#### Step 1.9: Enable Follower dataset in dataset.py

In `verl/utils/dataset.py`, **uncomment the Follower `RLHFDataset` class** (lines 375–483) and **comment out the Instructor `RLHFDataset` class** (around line ~220).

#### Step 1.10: Configure Follower training launcher

Edit `examples/qwen1.sh`:

```bash
MODEL_PATH=Qwen/Qwen2.5-7B-Instruct  # base model for first Follower
SAVE_PATH=/path/to/save/F_T1
DATA_PATH=data/Self-IF-Self-QWEN-7B/q_T1.parquet
```

Also update `examples/config1.yaml` to use `q_T1.parquet`:

```yaml
data:
  train_files: data/Self-IF-Self-QWEN-7B/q_T1.parquet
```

#### Step 1.11: Start vLLM verifier server

```bash
# Use the base model as the initial verifier
bash vllm_verifer.sh
```

#### Step 1.12: Launch Follower training

```bash
bash examples/qwen1.sh
```

The Follower learns to follow multi-constraint instructions by maximizing the reward from `instruction_compute_score()`.

**Output**: Follower checkpoint (e.g., `/path/to/F_T1/`).

### 5.4 Round T (T ≥ 2): Train Next Instructor

In round T, use the **latest Follower** (F_{T-1}) as the evaluator. This forces the Instructor to generate instructions that the current Follower cannot yet follow — creating a self-improving loop.

#### Step 2.1: Update vLLM servers to use Follower F_{T-1}

Replace the model in all three vLLM server scripts:

**`vllm_answer.sh`** — update model path and ensure it serves on port 55222:

```bash
# Replace MODEL_PATH with F_{T-1} checkpoint
MODEL_PATH=/path/to/F_{T-1}
```

**`vllm_filter.sh`** — update model path for port 55333:

```bash
MODEL_PATH=/path/to/F_{T-1}
```

**`vllm_judge.sh`** — update model path for port 55111:

```bash
MODEL_PATH=/path/to/F_{T-1}
```

Restart all three servers:

```bash
# Kill old processes and restart
bash vllm_answer.sh
bash vllm_filter.sh
bash vllm_judge.sh
```

> **Key principle**: The Instructor's training reward depends on how well the Follower follows the generated instructions. By swapping in the latest Follower, the Instructor learns to generate harder constraints.

#### Step 2.2: Enable Instructor reward in instruction_reward.py

In `examples/reward_function/instruction_reward.py`, **uncomment the Instructor reward block** and **comment out the Follower reward block**.

#### Step 2.3: Enable Instructor dataset in dataset.py

In `verl/utils/dataset.py`, **uncomment the Instructor `RLHFDataset` class** and **comment out the Follower `RLHFDataset` class**.

#### Step 2.4: Configure and launch Instructor training

```bash
MODEL_PATH=/path/to/I_{T-1}  # previous Instructor checkpoint
SAVE_PATH=/path/to/save/I_T
bash examples/qwen.sh
```

**Output**: New Instructor checkpoint I_T.

---

### 5.5 Round T (T ≥ 2): Train Next Follower

#### Step 2.5: Serve new Instructor I_T

```bash
# Update vllm_chuti.sh if needed (or it may already be configured)
bash data/Self-IF-Self-QWEN-7B/vllm_chuti.sh
```

#### Step 2.6: Generate new training data

```bash
cd data/Self-IF-Self-QWEN-7B
python chuti.py
```

This generates `q_T.parquet` (e.g., `q_T2.parquet`) containing instructions from the latest Instructor.

#### Step 2.7: Enable Follower reward in instruction_reward.py

In `examples/reward_function/instruction_reward.py`, **uncomment the Follower reward block** and **comment out the Instructor reward block**.

#### Step 2.8: Enable Follower dataset in dataset.py

In `verl/utils/dataset.py`, **uncomment the Follower `RLHFDataset` class** and **comment out the Instructor `RLHFDataset` class**.

#### Step 2.9: Update config1.yaml and qwen1.sh

Update `examples/config1.yaml`:

```yaml
data:
  train_files: data/Self-IF-Self-QWEN-7B/q_T.parquet
```

Update `examples/qwen1.sh`:

```bash
MODEL_PATH=/path/to/F_{T-1}  # previous Follower checkpoint
SAVE_PATH=/path/to/save/F_T
```

#### Step 2.10: Update verifier to use latest Follower F_T

Edit `vllm_verifer.sh` to use F_T as the verifier:

```bash
MODEL_PATH=/path/to/F_T
```

Restart the verifier:

```bash
bash vllm_verifer.sh
```

#### Step 2.11: Launch Follower training

```bash
bash examples/qwen1.sh
```

**Output**: New Follower checkpoint F_T.

  model: Qwen/Qwen2.5-7B-Instruct

  adv_estimator: grpo          # GRPO algorithm
  kl_coef: 1.0e-2

  actor:

  total_epochs: 1

## 7. Reward Functions

Both Instructor and Follower training use `instruction_reward.py`, but the active implementation switches based on which block is uncommented.

### 7.1 Instructor Training Reward

**Active block at the top of `instruction_reward.py`.**

Uses a three-stage verification pipeline via vLLM endpoints:

1. **Parse** generated instruction to extract constraints
2. **Filter** logically conflicting constraints using `FILTER_VLLM`
3. **Generate** answers for each constraint using `ANSWER_VLLM`
4. **Judge** constraint adherence using `JUDGE_VLLM`
5. **Return** score = (number of followed constraints) / (total valid constraints)

Higher score = Instructor generates instructions that the Follower can follow well.

### 7.2 Follower Training Reward

**Commented block at lines 1–~1040 of `instruction_reward.py` (uncomment to enable).**

Uses `instructions_registry` to directly evaluate each constraint:

1. **Parse** the ground truth `instruction_id_list`, `kwargs`, and `constraints` from the parquet file
2. **Instantiate** each constraint via `instructions_registry.INSTRUCTION_DICT[ids]`
3. **Call** `build_description()` with the stored `kwargs`
4. **Check** whether the Follower's response follows each constraint via `check_following()`
5. **Return** score = (number of followed constraints) / (total constraints)

Higher score = Follower successfully follows more constraints.

Each iteration makes the Instructor generate harder, more diverse constraints that the current Follower cannot yet follow, while the Follower improves its ability to follow complex multi-constraint instructions.
