Self-Evolving Reinforcement Learning

Rainier-rq1
Paper Code 5 Stars

Abstract

SEIF introduces an iterative self-training framework for instruction-following models, consisting of two mutually improving roles: the Instructor learns to generate diverse and challenging multi-constraint instructions, while the Follower learns to follow the instructions generated by the Instructor. The training process alternates between improving the Instructor and the Follower, enabling both models to progressively evolve over multiple rounds. By using the latest Follower as a judge, filter, and answer model during Instructor training, SEIF establishes a closed self-evolution loop that produces increasingly capable instruction-following models.

Overview

Instructor

Generates Instructions

Creates diverse, multi-constraint instructions from seed questions that challenge the current Follower model.

Follower

Follows Instructions

Trained to understand and satisfy the complex multi-constraint instructions generated by the Instructor.

Co-Evolution

Self-Improving

The latest Follower evaluates the next Instructor round, driving both models to improve together.

Iterative Training Workflow

1

Train Instructor IT

Use vLLM servers (ANSWER, FILTER, JUDGE) to train the Instructor. For T=1, use the base model as evaluator. For T≥2, use the previous Follower FT-1.

bash examples/qwen.sh
2

Generate Training Data

Serve the trained Instructor with vLLM and run constraint generation to produce qT.parquet for Follower training.

bash data/Self-IF-Self-QWEN-7B/vllm_chuti.sh && python chuti.py
3

Train Follower FT

Train the Follower using the generated qT.parquet. Initialize from FT-1 for T≥2.

bash examples/qwen1.sh

Reward Functions

Instructor Training

Uses three-stage verification pipeline:

  1. FILTER_VLLM: Detects logically conflicting constraints
  2. ANSWER_VLLM: Generates answers for valid constraints
  3. JUDGE_VLLM: Judges whether each constraint is followed
score = followed_constraints / valid_constraints

Follower Training

Uses instructions_registry to directly evaluate constraints:

  1. Parse instruction_id_list, kwargs, and constraints from parquet
  2. Instantiate each constraint using registry
  3. Check whether Follower's response satisfies each constraint
score = followed_constraints / total_constraints
